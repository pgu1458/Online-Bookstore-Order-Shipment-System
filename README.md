## 프로젝트 소개

5인 팀이 함께 만든 통합 유통 시스템입니다. 온라인 주문이 화면 위에서만 끝나지 않고 **실물 출고 장치까지 끊김 없이 연결**되는 흐름을 직접 구현하는 것이 목표였습니다.

전체 흐름은 — Flask 고객 홈페이지에서 도서를 검색·주문 → DB 재고 실시간 검증 → MFC 관리자가 출고 승인 → C# WinForms 하드웨어 제어 시스템이 컨베이어·서보를 구동해 실물 도서를 출고합니다. 저는 그중 **C# 하드웨어 제어 시스템 전체**와 **Flask 고객 홈페이지 전체 UI·기능**, 그리고 **XML/CSV 기반 설정·로그 관리**를 담당했고, 하드웨어 본체 제작에도 협업으로 참여했습니다.

## 주요 기능

###  1. C# WinForms 하드웨어 제어 시스템 (본인 단독)

실물 출고 장치를 직접 제어하는 데스크톱 프로그램입니다. MFC 관리자에서 출고가 승인된 주문이 들어오면, 이 프로그램이 컨베이어·서보·카메라를 조작해 도서를 정확한 위치까지 운반합니다.

**구현한 기능**

| 기능 | 내용 |
|------|------|
| 컨베이어 속도 제어 | 0~150 범위로 설정 변경, 정방향/역방향 선택 구동 |
| 영상 스트리밍 | 라즈베리파이 카메라 라이브 영상 수신 후 화면 출력 |
| 동작 결과 수신·표시 | 하드웨어 동작 완료 후 결과 메시지 수신·표시 |
| 장비 로그 | 컨베이어·서보·카메라 모든 동작을 시각과 함께 로그 패널에 출력 |
| 모터 상태 모니터링 | 컨베이어·서보의 현재 상태(정지/구동/속도) 실시간 표시 |
| **비상정지** | 단일 버튼으로 컨베이어·서보·전체 동작을 즉시 정지하는 안전 제어 |

**구현한 페이지**

|--------|------|

####  3D 호버 효과

도서 상세 페이지에서 책 표지에 마우스를 올리면 입체적으로 반응하도록 **3D 호버 효과**를 적용했습니다. CSS `transform: perspective` + `rotateX/rotateY`를 마우스 좌표에 따라 동적으로 계산해, 책 표지가 마우스 위치를 따라 미세하게 기울어지며 입체감을 만듭니다. 정적 이미지로 끝낼 수 있는 부분에 작은 디테일을 더해 UI 완성도를 끌어올린 부분입니다.

###  3. XML · CSV 기반 설정 + 이력 관리 (본인 단독)

하드웨어 동작 파라미터를 코드와 분리하고, 동작 이력을 체계적으로 기록하는 데이터 관리 구조를 설계했습니다.

#### XML — 동작 파라미터 분리

재고 슬롯 간격·딜레이·컨베이어 속도 같은 값들을 코드에 직접 박지 않고 `config.xml`로 분리해, **재컴파일 없이 값만 바꿔도 동작이 조정**되도록 했습니다. 현장에서 모터 캘리브레이션이 필요할 때 빠르게 대응할 수 있는 구조입니다.

| 파라미터 | 값 | 의미 |
|----------|-----|------|
| SlotGap (A→B) | **134** | 슬롯 A에서 B로 가는 스텝 간격 |
| SlotGap (B→C) | **134** | 슬롯 B에서 C로 가는 스텝 간격 |
| SlotGap (C→A) | **135** | 슬롯 C에서 A로 가는 스텝 간격 |
| Delay | 0.08s | 스텝 간 딜레이 |
| ConveyorSpeed | 70% | 컨베이어 기본 속도 |

슬롯 간격을 셋 다 같은 값(134)이 아니라 한 칸은 135로 둔 이유는, **물리적 슬롯 위치에 미세한 편차가 있어 실측 보정**한 결과입니다.

#### CSV — 동작 이력 기록

장비 동작 이력을 종류별로 분리해 CSV에 기록합니다.

| 파일 | 기록 내용 | 예시 |
|------|----------|------|
| `status_log.csv` | 장치 상태 점검 | `상태확인, 전체 장치 정상` |
| `conveyor_log.csv` | 컨베이어·서보·스텝모터 구동 | `정방향구동, 속도=70%, 시간=1000ms` |
| `video_log.csv` | 카메라 영상 시작·캡처·중지 | `캡처, capture_20260428_121026.png` |

이 구조로 어떤 시점에 어느 장치가 어떻게 동작했는지 시각과 함께 추적 가능합니다.

## ⚠ 한계 및 개선 방향

**자격증명 하드코딩**
초기 버전에 DB·라즈베리파이 SSH 접속 정보가 소스에 직접 들어 있어, 환경변수 또는 별도 설정 파일로 분리하는 작업이 필요합니다.

**동시성 — 낙관적 차감**
Flask 측 재고 차감은 조건부 UPDATE + rowcount 검사로 동시 주문 시 음수가 되지 않도록 막았지만, 명시적인 행 잠금(`SELECT ... FOR UPDATE`)은 사용하지 않았습니다. 트래픽이 더 커지면 락 전략이 필요합니다.

**XML 핫리로드 미지원**
`config.xml` 값을 바꿔도 프로그램을 재시작해야 반영됩니다. 파일 변경 감지(`FileSystemWatcher`)로 핫리로드를 추가하면 현장 캘리브레이션 효율이 더 좋아집니다.

**비상정지 — 소프트 단독**
현재 비상정지는 C# 프로그램에서 명령을 보내는 소프트 비상정지입니다. 펌웨어 레포(`Smart-Factory-Sorting-Line-Firmware`)에서 적용한 NC/NO 회로 이중화 방식이 산업안전 측면에서 더 안전합니다.

**카메라·하드웨어 단일 인스턴스 가정**
라즈베리파이 1대·C# 프로그램 1개 연결을 가정합니다. 다중 라인 확장 시 식별 ID와 재연결 처리가 필요합니다.
